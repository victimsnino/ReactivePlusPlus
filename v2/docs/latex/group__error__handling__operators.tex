requires \mbox{\hyperlink{conceptrpp_1_1constraint_1_1observable}{rpp\+::constraint\+::observable}}$<$std\+::invoke\+\_\+result\+\_\+t$<$TNotifier, std\+::exception\+\_\+ptr$>$$>$\\
auto rpp\+::operators\+::retry\+\_\+when (\begin{DoxyParamCaption}\item[{TNotifier \&\&}]{notifier}{}\end{DoxyParamCaption})}



If an error occurs, invoke {\ttfamily notifier} and when returned observable emits a value resubscribe to the source observable. If the notifier throws or returns an error/empty observable, then error/completed emission is forwarded to original subscription. 


\begin{DoxyParams}{Parameters}
{\em notifier} & callable taking a std\+::exception\+\_\+ptr and returning observable notifying when to resubscribe\\
\hline
\end{DoxyParams}
\begin{DoxyWarning}{Warning}
retry\+\_\+when along with other re-\/subscribing operators needs to be carefully used with hot observables, as re-\/subscribing to a hot observable can have unwanted behaviors. For example, a hot observable behind a replay subject can indefinitely yield an error on each re-\/subscription and using retry\+\_\+when on it would lead to an infinite execution.
