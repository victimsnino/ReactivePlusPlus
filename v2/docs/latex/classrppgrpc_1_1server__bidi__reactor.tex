class rppgrpc\+::server\+\_\+bidi\+\_\+reactor$<$ Request, Response $>$}
RPP\textquotesingle{}s based implementation for grpc server bidirectional reactor. 

To use it you need\+:
\begin{DoxyItemize}
\item create it via {\ttfamily new} operator
\item return it from bidirection method of Callback\+Service interface
\item to access values FROM stream you can subscribe to observable obtained via {\ttfamily reactor-\/\texorpdfstring{$>$}{>}get\+\_\+observable()} (same observable WOULD emit on\+\_\+completed in case of successful stream termination and on\+\_\+error in case of some errors with grpc stream)
\item to pass values TO stream you can emit values to observer obtained via {\ttfamily reactor-\/\texorpdfstring{$>$}{>}get\+\_\+observer()}
\end{DoxyItemize}

\begin{DoxyWarning}{Warning}
grpc server reactor have to finish manually, so it is expected that you call {\ttfamily on\+\_\+completed()} on reactor-\/\texorpdfstring{$>$}{>}get\+\_\+observer()
\end{DoxyWarning}
